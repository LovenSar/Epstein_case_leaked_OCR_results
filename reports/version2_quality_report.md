## 3. 路由模式与执行策略

### 3.1 V2 运行策略（实际使用）

- 使用 `--fast-scan`：单页单次主调用、关闭 refine，优先吞吐。
- `--router-votes 3` 在 fast-scan 下不参与多投票路由（由 fast path 接管）。
- 视频使用小批量多帧模式（`fast_video_batch_size`），并保留断点续跑。
- `--resume + --scan-cache`：支持断点续跑与目录缓存。

### 3.2 V2 图像路由逻辑（代码层）

- `process_single_image_fast()` 先走 `_program_route_image()` 的启发式判别：
  - 白底比例高且方差低 -> `DOC`
  - 亮度高且方差低 -> `DOC`
  - 其余 -> `VISION`
- `DOC` 走转录提示词，`VISION` 走详述提示词。
- 输出页头带 `## Page N (DOC|VISION)` 与 `[TAG:DOJ_CLEAN_V2]`。
